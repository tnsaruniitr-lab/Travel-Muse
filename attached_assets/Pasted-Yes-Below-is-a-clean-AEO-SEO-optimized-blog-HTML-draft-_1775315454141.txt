3) What makes this stronger for AEO / GEO / SEO

This structure is optimized because it does four things well:

Answers the main query early
The “Quick answer” section gives AI engines and search engines a clean extractable answer.
Uses natural query-matching subheads
Example:
where to stay in Oahu for a group trip
is Oahu good for a group trip with friends
what to plan in advance
Creates retrieval-friendly FAQ coverage
Good for Google, ChatGPT-style summarization, Perplexity-style citations, and snippet extraction.
Connects the article to product intent naturally
The CTA is not random. It fits the coordination problem the article is solving.
4) My direct recommendation

For GetTryps, I would use exactly this schema stack:

BlogPosting
FAQPage
BreadcrumbList
optional ItemList

I would not add unnecessary schema clutter like TravelItinerary unless you have multiple destination/day objects with richer structured travel metadata. For this use case, that usually adds complexity without much gain.
